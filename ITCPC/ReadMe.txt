Конвертор текстов Win <--> БК
Использование:
ITCPC -h (--help)
  Вывод этой справки.
ITCPC [-i<in_mode>] [-o<out_mode>] [-t<n>] <InputFile1.ext> {<InputFileN.ext> ...}
  -i<in_mode> (--input <in_mode>) - задать формат входного файла.
  Возможные режимы: a - автоопределение (по умолчанию, если параметр
      отсутствует). Не работает для БКшных файлов. Они опознаются как
      файл в KOI8R, и будут обработаны неправильно.
                    b - Basic ASC файлы. InputFile1.ext должен быть
      заголовочным .ASC файлом в .bin формате.
                    u - Unicode UTF-16LE или Unicode UTF-8, форматы
      корректно определяются даже без BOM.
                    w - Windows CP-1251
                    o - OEM CP-866
                    k - KOI8R
                    e - БКшный файл в формате Edasp
                    v - БКшный файл в формате Vortex

  -o<out_mode> (--output <out_mode>) - задать формат выходного файла.
  Возможные режимы: u - Unicode UTF-16LE (по умолчанию, если параметр
      отсутствует)
                    8 - Unicode UTF-8
                    w - Windows CP-1251
                    o - OEM CP-866
                    k - KOI8R
                    e - БКшный файл в формате Edasp
                    v - БКшный файл в формате Vortex
                    b - Basic ASC файлы (Имеет смысл только для листингов
      текстов на Бейсике).

  -t<n> (--tab <n>) - задать ширину табулятора для входного файла.
      Допускаются значения от 2 до 16, остальные игнорируются.
      Принятые значения по умолчанию: для файлов Windows tab = 4,
      для файлов БК tab = 8.

  -r (--recursive) - рекурсивный обход всех подкаталогов.

  { InputFile.ext } - список входных файлов. Допускается использование масок и
      задание путей, как абсолютных, так и относительных.
